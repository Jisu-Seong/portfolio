● 장점
- 특정 영화에 대한 영화 촬영지(여행지)를 즉각적으로 알 수 있음
- 여러 영화에 대한 경로를 자기가 직접 짜는 시간을 대폭 감소 시킬 수 있음
- 여러가지의 경로를 제공함으로써 여행자(유저)에게 다양한 선택권을 제공함
- 촬영지(여행지)마다 근처 숙소, 식당 정보를 제공하여 여행자(유저) 숙소 탐색 시간 단축
- 나의 경로 즐겨찾기 등록으로, 마음에 들었던 여행지를 다시 확인하고 여행할 수 있는 정보 제공으로 영화 선택, 촬영지 선택을 다시 처음부터 하지 않도록 편의성 제공

● 기술 스택
- java, jsp, mysql, HTML/CSS, javascript, maven, github, figma

● 학습 목표
- 3 tear architecture의 이해
- jsp, 서블릿 및 필터의 학습
- maven dependency 적용
- HTML/CSS, javascript 학습

● 담당 업무
- 경로 선택 화면 구성/로직 구현, 마이페이지 내 경로 확인

● ERD



● 개발 중점 사항
- 경로 저장을 위한 로직 구현
1. 중복 유무를 판별하는 DB 접근 필요
2. 새로운 경로 저장을 위한 DB 접근 필요
3. 상위 레이어 서비스 객체를 정의하고 커넥션 및 트랜잭션 제어 구현하여 원자성, 일관성 보장

● 이슈 사항 및 해결
- 영화의 목록이 많아 사용자에게 보여주는 컨텐츠 구성을 고려
1. 페이지 이동을 위해, 클릭 상호작용이 필요한 사용자 경험의 개선을 고려
2. 사용자의 페이지 스크롤 동작에 따라, 보고 있는 필요 영역만의 정보를 동적 렌더링하기 위해, 서버에 API 구현 후 비동기 통신을 활용한 Client-side Rendering 방식 도입

● 개선방안
- 경로 알고리즘이 위도, 경도 정보를 기반한 단순 직선 거리로만 계산하고 있음
> 공항과 숙소, 경로 간 이동시간, 촬영지에 머무르는 시간, 촬영지 경로 외 여행 시간, 촬영지의 운영 시간, 휴무일 등 다양한 요소들을 고려해야 함
> 과거 여행 기록과 관심 목록을 기반으로 계산하여 사용자에게 개인화된 경험을 제공할 수 있어야 함

● 장점
- 경매를 한 번도 해본 적 없어 실제 경매에 참여하기 부담스러운 사람도 경매 과정을 체험해볼 수 있음
- 회원이라면 누구나 경매 물품을 등록하거나 경매에 참여할 수 있음
- 실시간으로 경매 물품의 가격, 남은시간, 종료여부가 화면에 반영됨으로써 사용자에게 직관적인 인터페이스를 제공함

● 기술 스택
- java, mysql, quartznet, github, figma

● 학습 목표
- Mysql DB설계, DB 커넥션 응용

● 담당 업무
- 회원가입 및 로그인 로직, 메인 카테고리별 분류, 마이페이지 등록물품/낙찰 및 유찰물품 표시

● ERD



● 개발 중점 사항
- 실시간 정보 표시
> 개발에 앞서 경매 특성상 실시간으로 정보가 변경되므로 실시간으로 화면 정보가 갱신되게끔 하고자 회의하였고, 동시 작업이 필요하기에 멀티스레드 환경에 적합해야 했음
> 확장에 불리한 java 표준 라이브러리의 Timer 대신, 외부 라이브러리인 Quartz 스케줄러를 적용하여 1초마다 정보가 갱신되게끔 함으로써 사용자로 하여금 실제 경매와 흡사한 현장감을 제공함
- 정기 회의 및 소통
> 매일 아침 팀원들과 모여 진행상황을 보고하고 당일의 업무를 부여받아 수행하여 팀원 간의 소통을 중시함

● 이슈 사항 및 해결
- 초기 다량의 이미지를 포함한 물품 정보를 1초마다 가져와 갱신하니 서버가 굉장히 느려졌고, 일정 시간 이후 Heap 메모리의 부족 현상이 일어나 정상적인 프로그램 구동이 불가능했음
> 전체 회의 및 조사 결과 캐시를 사용하여 프로그램 내 갱신되는 정보와 DB에 있는 정보의 사이즈를 비교하여, 그 변경점만을 DB에 갱신하도록 하여 메모리 과부하를 방지하고, 서버 부하 해소.

● 개선방안
- 현재 캐시 활용을 위한 비교 방식은 길이 값만을 고려해, 차후 충돌 문제가 생길 가능성이 있음
> MD5와 같은 해시 함수를 이용할 시 입력값의 1비트만 변경되어도 결과값만을 비교하면 변경점을 감지할 수 있음
